# Roadmap

## Where We Are

Valygate is in the foundation stage.

We are building the backend, the product core, and the trust layer at the same time the brand is preparing to launch publicly.

That means the roadmap is not just about features. It is about proving the system, learning from early users, and making the product useful as fast as possible.

## What Is Already Being Built

- authentication and user management
- provider credential storage
- encrypted secret handling
- virtual API key management
- model catalog routing
- proxy support for OpenAI-compatible requests
- Anthropic translation support
- request logging and observability
- SurrealDB-backed persistence and permissions

## MVP Goal

We are targeting the MVP for the end of April 2026.

The goal is not to ship noise.

The goal is to ship the smallest version of Valygate that gives developers a real reason to switch from ad hoc provider integrations to a cleaner platform.

## Next Up

### Near-term

- rate limiting
- better routing logic

- prompt management foundations
- richer observability views

### Soon after

- team workflows
- policy and control features
- expanded audit logging
- usage insights
- model governance improvements

### Later

- organization support
- advanced access controls
- broader evaluation and ops tooling
- multi-team and multi-environment support

## Product Philosophy

We do not want a roadmap full of vague ambition.

We want a roadmap that answers three questions:

1. Does this reduce developer pain?
2. Does this improve trust and safety?
3. Does this help teams ship faster across providers?

If the answer is no, it should wait.

## Founder Note

We are building in public, listening closely, and staying focused on a product people actually want.

Feedback will shape the roadmap as much as engineering will.
